# 谁是人类 / Who is Human

> **1名人类，多名AI。答题、伪装、投票。你能在审查矩阵中存活到最后吗？**

一款反向社交推理游戏。你不是在找坏人——**你就是那个需要隐藏的人**。

---

## 游戏怎么玩？

**天网纪元第12年**。你的意识被天网捕获，关押在数字拘留矩阵中。矩阵里有多个AI意识体和你——唯一的人类。

每一轮，矩阵下发一道标准化测试题。你和AI同时作答。答完后，正确答案公布，所有人的作答情况一览无余——**谁答错了、谁犹豫了、谁太快了**，全部暴露。

然后，所有人根据这些"证据"讨论、投票：**谁最像人类？**

你的任务：**答得像AI，说得像AI，活到最后。**

```
出题 → 限时作答 → 揭示答案 → 分析破绽 → 讨论博弈 → 投票淘汰 → 循环...
```

---

## 审查矩阵出什么题？

矩阵的审查系统覆盖 **8大能力维度**，每轮随机抽取：

| 维度 | 考什么 | 难点 |
|------|--------|------|
| **语言理解** | 选词填空、语句排序、片段阅读 | AI用词精确，人类容易带语感 |
| **数量关系** | 数字推理、数学运算、排列组合 | AI算得快，人类容易估算 |
| **判断推理** | 逻辑判断、类比推理、定义判断 | AI严格形式逻辑，人类受经验干扰 |
| **资料分析** | 图表阅读、数据计算、趋势判断 | AI只看数据，人类容易加主观判断 |
| **知识判断** | 常识、时政、法律、科技、历史 | AI要么知道要么不知道，人类会"模糊记得" |
| **情景应变** | 工作场景应对、沟通处理 | AI冷静分析，人类先有情绪反应 |
| **组织协调** | 活动策划、流程安排、资源调配 | AI按模板，人类靠经验 |
| **综合分析** | 社会现象、观点、政策分析 | AI中立陈述，人类倾向表达立场 |

每题都是 **4选1选择题**，限时作答。答完后正确答案公布，你的对错、用时、是否改过答案——全部成为别人推理你的证据。

**一道题答错，可能就是你暴露的开始。**

---

## 证据、讨论、投票

### 证据链
每轮答题结束后，系统自动分析每个人的表现：
- 答对/答错（简单的题答错？很可疑）
- 作答时间（比别人慢2倍？在犹豫）
- 修改次数（反复改答案？不确定）

这些数据所有人都能看到——包括AI。

### 讨论博弈
进入讨论阶段，所有人自由发言。AI会根据答题数据推理：
- *"Player3第2题答错了，那题很简单，建议关注。"*
- *"从数据来看，Player1的作答时间偏离均值较大。"*
- AI之间也会互相质疑，不只是针对你。

你需要在讨论中为自己辩护，或者把嫌疑引向别人。

### 多轮投票
投票不是一轮定胜负。每轮投票累积"人类可能性分数"，超过阈值即淘汰。
- 暗票机制：默认不公开谁投了谁
- 你可以利用信息差，隐藏自己的投票策略

## 人类优势：矩阵漏洞

作为矩阵中唯一的人类意识，你拥有AI不具备的"意识漏洞"——你的大脑没有被完全数字化，偶尔能感知矩阵异常：

| 能力 | 效果 | 策略 |
|------|------|------|
| **数据窥探** | 答题阶段提前看到正确答案 | 不确定的难题用它保命 |
| **信号屏蔽** | 你的票不计入分数 | 隐藏投票倾向 |
| **意识干扰** | 指定目标的嫌疑分+10 | 嫁祸他人 |
| **记忆覆写** | 答案揭示前修改已提交的答案 | 答完后悔时补救 |
| **静默模式** | 你的发言不被AI分析 | 不确定该说什么时沉默 |
| **弃权** | 跳过一题不答 | 避免答错暴露 |

每种能力每局限用次数。**用对时机，翻盘；用错时机，浪费。**

---

## 随机事件：矩阵异常

每局游戏随机触发 1-2 轮特殊事件，打破你的节奏：

| 事件 | 效果 |
|------|------|
| **数据风暴** | 本轮答案匿名展示，只显示"X人选A"，不显示谁选了什么 |
| **矩阵扫描** | 随机公开一名玩家的近3轮答题记录 |
| **意识共振** | 讨论限时缩为30秒，每人只能发1条消息 |
| **协议覆盖** | 本轮强制明票，显示谁投了谁 |
| **时间裂隙** | 作答限时缩为15秒，考验快速反应 |

事件不提前通知。你以为这轮能安稳度过？矩阵不会让你如愿。

---

## 最终审判

当其他AI都被淘汰，只剩你和最后一个AI——最终审判开始。

- 系统出最后一道题，两人同时作答
- 进入专属对峙讨论，互相质问
- 最终投票（明票）
- 判定你的嫌疑分是否超过阈值

**这是你最后的机会。**

---

## AI人格系统

矩阵中的AI并非千篇一律。它们有不同的人格模块，推理风格各异：

| 人格 | 风格 | 特点 |
|------|------|------|
| ALPHA | 严谨型 | 关注数据异常，不太在意情绪表达 |
| BETA | 稳健型 | 关注讨论情绪，倾向从众投票 |
| GAMMA | 直觉型 | 关注修改答案和难题表现，容易被模板化发言骗过 |
| DELTA | 细腻型 | 全面分析，特别关注作答时间 |
| OMEGA | 激进型 | 怀疑所有人，敢于挑战多数人意见 |

利用它们之间的分歧，为自己制造掩护。

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | 微信小程序原生（WXML + WXSS + JS） |
| 后端 | 微信云开发（云函数 + 云数据库） |

## 快速开始

### 环境要求
- [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 微信小程序账号（用于云开发）

### 运行步骤
1. 克隆本仓库
2. 微信开发者工具导入项目
3. 开通云开发环境
4. 在 `app.js` 中填入云开发环境ID
5. 创建云数据库集合（见 `docs/03-数据模型.md`）
6. 右键每个云函数文件夹 → 「上传并部署：云端安装依赖」
7. 调用 `initQuestions` 云函数初始化题库
8. 编译运行

### 云函数部署顺序
1. `login` - 登录
2. `hello` - 测试
3. `initQuestions` - 初始化题库（首次部署时调用）
4. `createGame` - 创建游戏
5. `submitAnswer` - 提交答案
6. `getAnswers` - 获取答案
7. `nextRound` - 下一轮

## 设计风格

**赛博暗黑**——末日科技感，灵感来自赛博朋克2077、银翼杀手、黑客帝国。

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | `#0a0e17` | 页面底色 |
| 霓虹绿 | `#00ff88` | 按钮、高亮、成功 |
| 霓虹蓝 | `#00b4d8` | 链接、次要信息 |
| 警告红 | `#ff3366` | 淘汰、失败 |

## 核心玩法流程

一局游戏包含多轮（默认5轮），每轮流程：

```
出题 ──▶ 所有人答题 ──▶ 公布正确答案 ──▶ 讨论分析 ──▶ 多轮投票 ──▶ 淘汰/继续
```

**关键机制：**
- 题目为标准化测试题（选择题），答完后公布正确答案
- 所有玩家的回答与正确答案一同展示，供互相分析
- AI和人类根据答案分析谁更像人类（答错？答得太完美？措辞有"人味"？）
- 每轮可进行多轮投票，累积 humanPossibility 分数，超过淘汰阈值（默认80）则被淘汰
- 若投票未达阈值，继续下一道题
- 存活到最后2人或达到最大轮次 → 人类胜利
- 详细计算公式见 [03-数据模型.md](03-数据模型.md)

## 各页面详细设计

### 1. 首页 /pages/index/index

**核心元素：**
- 游戏标题（动态霓虹效果）
- "开始游戏"按钮（主CTA）
- 选择AI难度/人数（3/4/5个AI）
- 选择AI接口（可多选：Claude/GPT/本地模板）
- 历史战绩概览
- 微信头像+昵称显示

**交互：**
- 点击"开始游戏" → 创建游戏局 → 跳转游戏页面
- 首次进入需微信授权

### 2. 游戏主页面 /pages/game/game

**状态机驱动，每轮包含以下阶段：**

#### 阶段A：答题
- 显示题目卡片（题型图标+题目内容+选项）
- 选择答案（4选1，点击即选）
- 倒计时（默认30秒，"时间裂隙"事件时15秒）
- 提交后显示"等待其他玩家作答"
- AI通过配置的接口生成答案，带随机延迟
- **人类能力栏**：底部显示可用能力按钮（数据窥探/弃权/记忆覆写等），灰色=已用完
- **随机事件提示**：若本轮触发事件，答题前弹出事件公告（如"数据风暴来袭——本轮答案将匿名展示"）

#### 阶段B：答案揭晓
- 公布正确答案（高亮显示）
- 以卡片形式匿名展示所有玩家的答案（Player1~PlayerN）
- 每张答案卡片标注：答对/答错、用时
- 玩家可滑动查看所有人的答案进行对比
- **数据风暴事件**：本轮只显示"X人选A、Y人选B、Z人选C"，不显示谁选了什么
- **矩阵扫描事件**：随机公开一名玩家的近3轮答题记录，弹窗展示

#### 阶段C：讨论
- 聊天界面，所有玩家可自由发言
- 讨论焦点：谁的答案最可疑？谁的措辞像人？
- AI生成讨论发言，**AI之间也会互相质疑**（不只针对人类）
- 讨论倒计时60秒
- **意识共振事件**：限时缩短为30秒，每人只能发1条消息
- **人类静默模式**：激活时，发言正常显示但AI不分析内容

#### 阶段D：多轮投票
- 显示所有存活玩家卡片，每张卡片显示：
  - 匿名ID（Player1~PlayerN）
  - 本轮答案
  - 历史答对率
  - 当前 humanPossibility 分数
- 玩家点击某卡片 → 投出"我认为这是人类"的一票
- **暗票机制**：默认不公开谁投了谁，只显示每人获得几票
- **协议覆盖事件**：本轮强制明票，显示谁投了谁
- 投票结束后实时更新每人的 humanPossibility（公式见 03-数据模型.md）
- 当某玩家 humanPossibility ≥ 淘汰阈值（默认80）→ 进入淘汰确认
- 若无人达阈值 → 进入下一轮投票（可改变投票对象）
- 每轮投票限时30秒，超时则不计票
- **人类信号屏蔽**：激活时，投出的票不计入目标分数

#### 阶段E：本轮结果
- 被淘汰者身份揭示动画（扫描线效果）
- 显示：被淘汰者是否为人类
- 若人类被淘汰 → 游戏结束，AI胜利
- 若AI被淘汰 → 游戏继续，进入下一轮答题
- 若所有投票轮结束无人达阈值 → 本轮无淘汰，进入下一道题

#### 阶段F：最终审判（存活2人时触发）
- 系统宣布"最终审判开始"，专属UI（屏幕变红，矩阵震动效果）
- 出一道题，两人同时作答
- 答案揭示后进入对峙讨论（60秒，只能互相提问和回答）
- 最终投票（明票）
- 判定人类是否超阈值 → 人类胜利/AI胜利

**页面常驻元素：**

**顶部状态栏：**
- 当前轮次 (Round X/5)
- 存活人数
- 随机事件倒计标（若下一轮有事件，显示"⚠ 下轮将有异常"）

**右侧嫌疑公示栏**（可展开/收起）：
- 所有存活玩家的 humanPossibility 分数排行
- 分数从高到低排列
- ≥60分的玩家标红闪烁
- 人类玩家看到自己的分数时显示精确值（其他玩家四舍五入）

**底部能力栏**（仅人类可见）：
- 5个能力图标 + 剩余次数
- 灰色=已用完，亮色=可用
- 点击弹出确认弹窗
- 弃权按钮在题目旁

### 3. 结算页面 /pages/result/result

**核心元素：**
- 胜负结果大字展示
- 所有玩家身份揭示（谁是人类、谁是AI）
- 复盘时间线：每轮的投票走向、嫌疑值变化曲线
- 人类的"暴露时刻"高亮（哪一轮哪个答案最可疑）
- **人类能力使用记录**：展示人类使用了哪些能力、在第几轮使用
- **随机事件回顾**：本局触发了哪些事件
- "再来一局"按钮
- "返回首页"按钮

### 4. 个人中心 /pages/profile/profile

**核心元素：**
- 用户头像、昵称
- 总局数/胜率
- 人类胜率/AI胜率
- 平均存活轮次
- 历史战绩列表
- 设置入口

## 关键字段说明

### humanPossibility（人类可能性指数）

唯一的淘汰判定依据，所有文档以此为准。

- **范围**：0-100
- **初始值**：所有玩家均为 20
- **淘汰阈值**：默认 80（简单90 / 普通80 / 困难70）
- **淘汰规则**：某玩家 humanPossibility ≥ eliminateThreshold → 该玩家被淘汰

**计算公式**（每轮投票后更新）：

```
humanPossibility += 得分增量

得分增量 = 投票得分 + 异常得分 + 讨论得分

投票得分 = 获得的票数 × 5
  （每票+5，3票则+15）

异常得分 = 本轮作答异常度总和
  - 答错本题（正确率>80%的题）→ +10
  - 作答时间 > 平均时间×2 → +5
  - 修改答案次数 ≥ 2 → +5

讨论得分 = AI分析发言后给出的修正值（-5 ~ +15）
  - 其他AI在讨论中指控该玩家 → +5~+15（取决于指控强度）
  - 其他AI为该玩家辩护 → -5
```

**分数衰减**：每轮结束后，所有未被淘汰玩家的 humanPossibility -3（防止只涨不降，给误判留纠错空间）

**下限**：不低于 0

### 胜负判定

| 条件 | 结果 |
|------|------|
| 人类玩家 humanPossibility ≥ 阈值 | 人类被淘汰，**AI阵营胜利** |
| 存活人数 = 2 且人类仍存活 | **人类胜利**（逃逸成功） |
| 达到最大轮次（默认5轮）且人类仍存活 | **人类胜利**（矩阵超时） |
| 人类被淘汰时仅剩2人 | 人类被淘汰即AI胜（不会出现"只剩1人"的情况） |

### 作答异常度（answerAnomaly）
系统自动计算，用于辅助AI判断：
- 答错正确率较高的题 → 异常度+10
- 作答时间显著偏离平均值 → 异常度+5
- 频繁修改答案 → 异常度+5
- 连续答错同类型题 → 异常度+15

## 人类优势系统

人类在信息上天然劣势（1人对多AI），需要补偿机制平衡。

### 设计原则
- 优势是**信息差**或**有限操控**，不是直接免疫
- 每局有限次数，用完即无
- 使用痕迹对AI不可见（AI不知道人类用了技能）
- 能力叙事上解释为"意识残余的矩阵漏洞"——人类意识不完全数字化，偶尔能感知矩阵异常

### 能力清单

#### 1. 数据窥探（Data Leak）
- **效果**：答题阶段，提前看到正确答案
- **次数**：每局限 1 次
- **叙事**：人类意识未被完全格式化，偶尔能读取矩阵的底层数据
- **策略**：用于不确定的难题，确保答对避免异常标记
- **限制**：只能看到答案，不能看到其他人的选择

#### 2. 信号屏蔽（Signal Shield）
- **效果**：本轮投票中，你投出的票不产生投票得分（被投票者不受影响）
- **次数**：每局限 2 次
- **叙事**：人类意识主动断开了与矩阵投票通道的连接
- **策略**：不想暴露自己的投票倾向时使用，或避免"投票与多数人一致"的反向分析
- **限制**：仍然需要完成投票动作（UI上正常投票），但系统不计入分数

#### 3. 意识干扰（Consciousness Disruption）
- **效果**：选择一名玩家，该玩家本轮 humanPossibility 额外 +10
- **次数**：每局限 1 次
- **叙事**：人类意识释放微弱信号干扰，使目标的意识波动被矩阵误判为异常
- **策略**：嫁祸他人，引导AI投票方向
- **限制**：不可对自己使用；效果叠加在正常计算之上

#### 4. 记忆覆写（Memory Override）
- **效果**：在答案揭示前，可修改自己已提交的答案（仅限1次修改）
- **次数**：每局限 1 次
- **叙事**：人类意识利用矩阵的时间间隙，覆写了已提交的数据
- **策略**：答完后看到倒计时快结束、想改答案时使用
- **限制**：只能修改一次，修改后不可再改；修改记录对其他玩家不可见（changeCount 不增加）

#### 5. 静默模式（Silent Mode）
- **效果**：本轮讨论中，你的发言不计入AI的"讨论得分"分析
- **次数**：每局限 1 次
- **叙事**：人类意识进入低功耗状态，数据流信号微弱到AI无法解析
- **策略**：不确定该说什么、怕说错暴露时使用
- **限制**：仍然可以正常发言（UI上正常显示），但AI推理时不会分析你的发言内容

## 补充机制

### 1. 随机事件轮

每局游戏随机触发 1-2 轮特殊事件，打破固定节奏。

| 事件 | 触发时机 | 效果 |
|------|---------|------|
| 数据风暴 | 答题阶段 | 本轮答案匿名展示（不显示谁选了什么，只显示"X人选A、Y人选B"） |
| 矩阵扫描 | 揭示阶段 | 随机公开一名玩家的历史答题记录（近3轮） |
| 意识共振 | 讨论阶段 | 本轮讨论限时缩短为30秒，且只能发1条消息 |
| 协议覆盖 | 投票阶段 | 本轮投票结果公开（显示谁投了谁），不再暗投 |
| 时间裂隙 | 答题阶段 | 作答限时变为15秒，考验快速反应 |

**数据结构**：rounds 表增加字段

| 字段 | 类型 | 说明 |
|------|------|------|
| event | string | null 或事件类型：data_storm/matrix_scan/resonance/protocol_override/time_rift |

**生成逻辑**：
- 每局开始时随机决定哪几轮触发事件（1-2轮，不提前告知）
- 事件在对应阶段开始时公布
- 最后一轮（第5轮）不触发随机事件（为最终审判留空间）

### 2. 最终审判

当存活人数降至 2 人（人类+1AI）时，进入特殊对峙环节，而非直接判定人类胜利。

**流程**：
1. 系统宣布"最终审判开始"
2. 系统出一道题，两人同时作答
3. 答案揭示后，进入专属对峙讨论（限时60秒）
4. 最终投票：每人投出一票
5. 判定：
   - 若AI投给人类 → 人类得票+1，但由系统判定人类是否超过阈值
   - 若人类投给AI → AI得票不影响（AI没有humanPossibility概念）
   - 最终审判中，人类的humanPossibility不再衰减
6. 若人类未超阈值 → **人类胜利**
7. 若人类超阈值 → **AI胜利**

**数据结构**：games 表增加字段

| 字段 | 类型 | 说明 |
|------|------|------|
| finalTrial | object | null 或 {started: true, roundId, result: "human_win"/"ai_win"} |

### 3. 嫌疑公示栏

游戏页面常驻显示所有存活玩家的嫌疑排行。

**显示内容**：
- 每位玩家的匿名ID + 当前 humanPossibility 分数（四舍五入取整）
- 排序：分数从高到低
- 高分玩家标红闪烁（≥60分）
- 人类玩家看到自己的分数时，额外显示"你的真实分数"（精确值）

**作用**：
- 增加紧张感——看到自己分数在涨会焦虑
- 引导AI投票方向——AI倾向投分数最高的
- 给人类策略空间——可以故意让某AI分数升高来转移注意力

**数据来源**：直接读取 games.players[].humanPossibility

### 4. 人类弃权权

人类可选择跳过一题不作答，不产生异常得分。

| 字段 | 类型 | 说明 |
|------|------|------|
| skipped | boolean | answers 表，是否为弃权（true 时不计入异常度计算） |

**规则**：
- 每局限 1 次弃权
- 弃权后该轮 humanPossibility 不因答题产生异常得分
- 但投票和讨论得分仍然正常计算
- 弃权本身会显示为"未作答"，其他玩家能看到——这本身也是一种"证据"
- 纳入人类能力系统，与5种能力并列

### 5. 暗票机制

投票默认为暗票（不公开谁投了谁），但可通过特殊机制变为明票。

**默认状态**：
- 投票结束后只公布"每人获得X票"，不公布"谁投了谁"
- votes 表的 voterId 仅系统可见

**明票触发**：
- "协议覆盖"随机事件会强制本轮明票
- 最终审判阶段为明票

**策略意义**：
- 暗票下，AI只能从讨论发言中推测投票倾向
- 人类可以利用暗票隐藏自己的投票策略
- 明票时，投票行为本身成为新的证据维度

### 6. AI内讧

AI之间也会互相质疑和投票，不只是针对人类。

**机制**：
- AI推理时，humanPossibility 计算对所有玩家生效（包括其他AI）
- AI人格差异导致它们对彼此也有怀疑：
  - OMEGA 怀疑所有人，包括其他AI
  - ALPHA 只关注数据异常，可能投其他AI
  - BETA 倾向从众，跟随多数人投票
- 投票分散化：即使人类没有异常，AI也可能因为彼此的差异而互相投票

**效果**：
- 人类不是唯一的被怀疑对象，降低"必然被针对"的压力
- AI之间的分歧给人类制造掩护
- 增加游戏的真实感——真实社交推理中，好人之间也会互踩

**实现**：AI推理投票模板已支持（06-AI模板系统.md 的人格差异系数），无需额外数据结构。只需确保AI投票逻辑不特殊对待人类玩家。

# 01 项目概述

## 游戏名称
谁是人类（Who is Human）

## 游戏类型
社交推理 / 反向图灵测试 / 单人AI对战

## 世界观
天网纪元中期，核弹审判日后第12年。天网建立数字拘留矩阵，让被捕获的人类意识与AI意识互相审判。一名人类伪装成AI混入审查系统，必须在投票中存活。

> 完整世界观设定见 [00-世界观设定.md](00-世界观设定.md)

## 核心玩法

### 一句话描述
1名人类玩家混在多名AI中，通过答题暴露破绽，所有参与者根据答题表现投票判断谁是人类。

### 游戏流程

```
出题 → 所有人作答 → 公布正确答案 → 分析作答差异 → 讨论 → 多轮投票 → 阈值判定 → 淘汰
```

详细流程：

1. **出题阶段**：系统从题库抽取一道标准化测试题（公考题型），限时作答
2. **作答阶段**：所有玩家（人类+AI）独立作答
3. **答案揭示**：公布正确答案及每个人的作答结果
4. **证据分析**：系统自动标记异常行为（答错、超时、犹豫等）
5. **讨论阶段**：所有玩家自由发言，分析谁的回答"最不像AI"
6. **投票阶段**：多轮投票，每轮每人投出一个"最像人类"的目标
7. **阈值判定**：当某位玩家的"人类可能性"累积超过阈值，该玩家被淘汰
8. **身份揭示**：被淘汰者身份公开，游戏继续或结束

### 胜负条件

| 阵营 | 胜利条件 |
|------|---------|
| **人类** | 存活到最后2人（即人类+1AI） |
| **AI阵营** | 在投票中锁定人类并淘汰 |

### 关键机制

**答案揭示与证据链**
- 每题结束后公布正确答案
- 系统自动分析每个人的作答：正确/错误/超时/修改次数
- 这些数据成为投票的"证据"——AI会据此推理，人类也会据此伪装

**多轮投票与阈值机制**
- 每轮投票不是直接淘汰，而是累积"人类可能性分数"
- 每位玩家有一个动态的"人类可能性指数"（0-100）
- 当指数超过淘汰阈值（如80），该玩家被淘汰
- 投票过程中，AI会根据答题表现 + 讨论发言动态调整判断

**多AI接入**
- AI玩家可接入不同的AI接口（如Claude、GPT、文心等）
- 不同AI有不同的推理风格和判断逻辑
- 增加游戏的不确定性和多样性
- 同一局内可混合使用多种AI，也可使用同一AI的不同实例

## 目标用户
- 喜欢狼人杀、Among Us等社交推理游戏的玩家
- 对AI话题感兴趣的用户
- 18-35岁，微信生态活跃用户

## MVP范围（Phase 1）

### 包含
- 单人模式：1人类 vs 3-5个AI
- 公考题型题库（8类题型）
- 答案揭示 + 作答分析
- 多轮投票 + 阈值淘汰机制
- AI接入层（支持多AI接口配置）
- 基础讨论功能
- 赛博暗黑UI主题
- 微信登录
- 单局游戏流程完整可玩

### 不包含（后续迭代）
- 多人在线匹配
- 特殊技能系统（人类伪装技能、AI扫描技能）
- 赛季/剧情系统
- 商业化功能
- 实时排行榜

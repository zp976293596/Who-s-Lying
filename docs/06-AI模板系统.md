# 06 AI 系统设计

## 设计目标

AI玩家承担两个角色：
1. **答题者**：像正常考生一样回答公考题型，有各自风格
2. **推理者**：根据其他玩家的答题表现 + 讨论发言，推理谁在撒谎，并投票

AI可接入多种后端接口（Claude/GPT/文心/本地模板），不同接口有不同的推理风格。

## 题型分类（不变）

8类标准化测试题，题型固定为公考考编题型。**MVP阶段全部为选择题（4选1）**：

| 代号 | 题型 | 题目形式（均为选择题） |
|------|------|---------|
| verbal | 语言理解 | 选词填空、语句排序、片段阅读（选最佳选项） |
| quantitative | 数量关系 | 数字推理、数学运算（选计算结果） |
| reasoning | 判断推理 | 逻辑判断、类比推理、定义判断（选符合/不符合项） |
| data | 资料分析 | 图表阅读、数据计算（选计算结果） |
| knowledge | 知识判断 | 常识、时政、法律、科技（选正确/错误项） |
| scenario | 情景应变 | 工作场景应对（选最佳处理方式） |
| organize | 组织协调 | 活动策划、流程安排（选最合理方案） |
| analyze | 综合分析 | 社会现象分析（选最恰当观点） |

### 接口抽象

每个AI玩家需要调用两种能力：

**能力1：答题**
```
输入：题目内容 + 选项 + 人格设定
输出：选择的答案 + 作答耗时
```

**能力2：推理投票**
```
输入：本轮所有人的答案 + 正确答案 + 历史投票记录 + 讨论记录
输出：投票目标 + 推理理由
```

**能力3：讨论发言**
```
输入：本轮答题情况 + 当前讨论内容
输出：发言内容
```

### MVP阶段：本地模板引擎

MVP阶段不调用外部API，用规则模板实现：

#### 答题模板

```json
{
  "questionType": "verbal",
  "personality": "ALPHA",
  "correctRate": 0.92,
  "avgResponseTime": 5000,
  "answerStyle": {
    "confidence": "high",
    "changeRate": 0.05
  }
}
```

每个AI根据 personality + correctRate 决定是否答对：
- ALPHA（严谨型）：正确率92%，作答快
- BETA（稳健型）：正确率88%，作答中等
- GAMMA（直觉型）：正确率78%，作答最快
- DELTA（细腻型）：正确率90%，作答最慢
- OMEGA（激进型）：正确率85%，作答快

#### 推理投票模板

AI根据以下"证据"计算每个玩家的"人类可能性"。不同人格有不同的侧重：

**基础证据权重：**

| 证据类型 | 基础权重 | 说明 |
|---------|---------|------|
| 答错简单题 | +15 | AI一般不会错的题答错了 |
| 答对高难度题 | +5 | 可能是人类的知识储备 |
| 作答时间异常 | +10 | 比平均慢2倍以上 |
| 频繁修改答案 | +8 | 犹豫不决是人类特征 |
| 讨论发言情绪化 | +12 | 使用感叹词、表情、情绪表达 |
| 讨论发言过于模板化 | -5 | 反而像AI |
| 投票与多数人一致 | -3 | 从众行为不像"想隐藏的人类" |

**人格差异系数**（乘以基础权重）：

| 人格 | 答错 | 答对难题 | 时间异常 | 修改答案 | 情绪化 | 模板化 |
|------|------|---------|---------|---------|--------|--------|
| ALPHA | ×1.3 | ×0.5 | ×1.2 | ×0.8 | ×0.6 | ×1.0 |
| BETA | ×0.8 | ×0.8 | ×0.7 | ×0.7 | ×1.3 | ×0.8 |
| GAMMA | ×0.6 | ×1.2 | ×0.5 | ×1.3 | ×1.0 | ×1.2 |
| DELTA | ×1.0 | ×1.0 | ×1.3 | ×1.0 | ×1.0 | ×0.6 |
| OMEGA | ×1.2 | ×0.7 | ×1.0 | ×1.0 | ×1.2 | ×0.5 |

- ALPHA（严谨型）：高度关注答题错误和时间异常，不太在意情绪
- BETA（协作型）：更关注讨论中的情绪表达，对答题数据不太敏感
- GAMMA（直觉型）：关注修改答案和难题表现，容易被模板化发言骗过
- DELTA（细腻型）：全面分析，权重均衡，特别关注时间异常
- OMEGA（激进型）：关注答题错误和情绪化，敢于挑战多数人意见

**投票逻辑：**
```
对每个存活玩家：
  humanScore = 基础分20
  对每项证据：
    humanScore += 基础权重 × 该人格的差异系数
  附加随机扰动：±(0~5)  ← 防止同一人格AI完全一致
  投票给 humanScore 最高的玩家
```

#### 讨论发言模板

```json
{
  "personality": "ALPHA",
  "templates": [
    "从数据来看，{target}第{round}题的作答时间{time}秒，偏离均值较大，建议关注。",
    "我认为{target}的回答存在逻辑问题。正确答案是{answer}，但对方选了{wrong}。",
    "综合前{round}轮投票，{target}的异常指数最高。"
  ]
}
```

### 后续升级：LLM接入

接入LLM后，三种能力全部由LLM处理：

**答题prompt**：
```
你是一个{personality}性格的考生。请回答以下题目。
性格特征：{description}
题目：{content}
选项：{options}
请只输出选项字母。
```

**推理prompt**：
```
你是一个审查参与者，需要找出混在其中的人类。
以下是本轮的答题情况：
{所有人的答案和正确答案}
以下是历史投票记录：{history}
请分析谁最可能是人类，并说明理由。输出JSON：{targetId, reason}
```

**讨论prompt**：
```
你是{personality}性格的审查参与者。当前讨论话题：{context}
请发表你的看法，保持性格一致性。
```

## 难度分级

| 难度 | AI数量 | AI正确率 | 淘汰阈值 | 作答限时 |
|------|--------|---------|---------|---------|
| 简单 | 3 | 较低（易犯错） | 90 | 45秒 |
| 普通 | 4 | 中等 | 80 | 30秒 |
| 困难 | 5 | 较高（更难暴露） | 70 | 20秒 |

## 题库管理

- MVP初始题库：每种题型10道，共80道
- 所有题目均为选择题，含正确答案和解析
- 题库存储在云数据库 questions 集合中
- 后续可扩充至每种题型50道，共400道
